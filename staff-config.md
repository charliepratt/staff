## Project

- **Name**: TypeyTypey
- **Type**: software (web app)
- **Description**: A writing application for all types of writers — screenplays, stage plays, novels, blogs, and more. Emphasizes rock-solid formatting that matches industry standards (e.g., proper screenplay format), supreme readability, and a live in-context preview (inspired by Vellum) rather than Markdown-based editing. Designed to feel approachable for everyday writers, not just power users.
- **Industry/Domain**: Creative writing tools
- **Target Audience**: Writers of all kinds — aspiring, hobbyist, and working writers who want a clean, approachable tool. Not targeting deep power users (Scrivener crowd) but rather the general population getting into writing across formats.
- **Tech Stack**: Web-based (TBD — early exploration)
- **Stage**: idea

## Business Context

- **Revenue Model**: TBD — exploring options
- **Key Competitors**: Scrivener, Vellum, Final Draft, Ulysses, and many others
- **Differentiator**: No Markdown — WYSIWYG with live in-context preview like Vellum. Approachable for non-technical writers. Rock-solid format compliance across writing types (screenplay, novel, blog, etc.). Does not host user content — connects to user's own cloud storage (Dropbox, Google Drive, etc.) by default, with local save as fallback.

## Key Principles

- **No content hosting** — user data lives on their own storage, never on our servers
- **Cloud-first backup** — connects to Dropbox, Google Drive, etc. so writers never stress about losing work
- **Local save available** — always an option, but cloud sync is the default experience
- **Format fidelity** — each writing type renders in its proper industry-standard format
- **Live preview** — see your work as it will look, not as raw markup
